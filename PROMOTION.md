# 告别收藏吃灰：一键把知乎好文存成 Markdown

你有没有这样的经历——

在知乎刷到一篇极好的技术文章，顺手收藏，想着以后慢慢看。半年过去，打开收藏夹，发现「该内容已被作者删除」。

或者你想把知乎上的回答搬进 Obsidian 做笔记，手动复制粘贴，格式全乱了，图片还丢了，代码块变成一坨纯文本。

如果你也被这些问题困扰过，这个小工具也许能帮到你。

---

## 知乎文章下载器：它能做什么

这是一个 Chrome 扩展，功能用一句话概括：**把知乎内容一键导出为高质量的 Markdown 文件**。

不需要登录，不需要 API Key，不需要任何配置。打开知乎页面，点一下浮动按钮，文章就到手了。

### 支持五种知乎内容

- **专栏文章** — 你关注的大V写的长文
- **问题回答** — 某个问题下的精彩回答
- **问题页面** — 整个问题及其所有回答一起打包
- **想法/动态** — 知乎的短内容
- **收藏夹** — 没错，整个收藏夹一键全部导出

### 转换质量远超复制粘贴

这不是简单的文本复制。导出的 Markdown 会完整保留：

- LaTeX 数学公式（行内 `$...$` 和块级 `$$...$$`）
- 带语法高亮的代码块
- 完整的表格结构
- 脚注引用
- 知乎特有的链接卡片和视频占位符

如果你用 Obsidian、Typora、Notion 等工具管理知识库，导出的文件可以直接使用，几乎不需要二次编辑。

### 图片不会丢失

勾选「下载图片」选项后，工具会自动下载文章中的所有图片，和 Markdown 文件一起打包成 ZIP。图片链接会自动替换为本地相对路径，离线也能正常查看。

---

## 收藏夹批量导出：真正解决「收藏吃灰」

这可能是最实用的功能。

打开任意知乎收藏夹页面，点击浮动按钮，工具会自动通过 API 分页获取收藏夹中的**全部内容**，逐篇转换为 Markdown，最终打包成一个 ZIP 文件。

导出的 ZIP 里还会自带一份 README.md 目录索引，按编号列出所有文章，点击即可跳转。

```
我的技术收藏.zip
├── README.md                    # 目录索引
├── 001_深入理解Go调度器-张三.md
├── 002_Redis持久化机制详解-李四.md
├── 003_TCP三次握手的本质-王五.md
├── ...
└── images/
    ├── 001_001.jpg
    ├── 002_001.png
    └── ...
```

收藏了 200 篇文章？没问题，点一下，等它跑完，全部到手。

## 使用方式

1. 安装扩展后，打开任意知乎页面
2. 页面右下角会出现一个小按钮（可以拖到你喜欢的位置）
3. 点击展开面板，确认内容信息
4. 按需勾选「Front Matter」和「下载图片」
5. 点击下载

就这么简单。没有注册流程，没有付费墙，没有使用次数限制。

## 关于隐私

作为一个开发者，我自己也很在意扩展的权限问题，所以这个工具的权限做到了最小化：

- 仅申请 `activeTab` 权限，只在你主动操作时读取当前页面
- 只运行在知乎域名下，不会注入任何其他网站
- 所有数据处理都在浏览器本地完成，不会上传到任何服务器
- 完全开源，代码透明可审查
